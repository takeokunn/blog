# カーステン・ドミニク（Carsten Dominik）の功績

-   2003年の最初のリリース
-   天文学者としての背景と「メモを取る」ニーズ
-   「人間にも機械にも読みやすい」という設計理念


# Org-modeの設計原則

-   プレーンテキストの永続性
-   最小限の構文、最大限の表現力
-   段階的な複雑さ（Simple things simple, complex things possible）


# Markdownとの比較

-   標準化の違い（CommonMark vs 単一実装）
-   メタデータ表現力の違い（YAML frontmatter vs PROPERTIES）
-   拡張性の違い（HTMLベース vs Elisp統合）


# プレーンテキストにおけるデータベース的アプローチ

-   プロパティによる構造化データ
-   タグによる分類
-   リンクによる参照
