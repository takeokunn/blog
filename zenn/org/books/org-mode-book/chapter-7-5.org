* corfu: 次世代の補完UI

- corfuとは
  - companyの軽量な代替
  - child-frameベースのUI
  - Emacs組み込みcompletion-at-pointとの統合
- 基本設定
  - corfu-auto（自動補完）
  - corfu-cycle（循環選択）
  - corfu-quit-no-match
- Org-modeでの活用
  - org-roam補完
  - リンク補完
  - タグ・プロパティ補完

* cape: completion-at-point extensions

- capeの役割
  - 補完ソースの拡張
  - 複数バックエンドの統合
- 主要な補完関数
  - cape-file（ファイルパス）
  - cape-dabbrev（動的略語）
  - cape-keyword（キーワード）
  - cape-elisp-block（Elispブロック）
- corfuとの連携
  - completion-at-point-functions
  - cape-super-capf
- Org-mode用の設定例

* textlint: 日本語文書の品質管理

- textlintとは
  - Node.jsベースの文書リンター
  - プラガブルなルールシステム
  - 日本語対応が充実
- セットアップ
  - npm install -g textlint
  - .textlintrc の設定
  - Emacs連携（flycheck-textlint / flymake-textlint）
- 推奨ルール
  - textlint-rule-preset-ja-technical-writing
  - textlint-rule-preset-jtf-style
  - textlint-rule-no-doubled-joshi
  - textlint-rule-sentence-length
- Org-modeでの利用
  - org-src-modeでの動作
  - エクスポート前のチェック
  - 特定ブロックの除外設定

* flycheck/flymake + textlint

- flycheck-textlint
  - インストールと設定
  - org-modeでの有効化
  - カスタムルールの追加
- flymake-textlint
  - Emacs標準flymakeとの統合
  - 設定例
- リアルタイムチェックの注意点
  - パフォーマンス考慮
  - 一時ファイルの扱い

* 補完とリンティングの統合

- 執筆ワークフローへの組み込み
- org-capture時の補完設定
- エクスポート前の自動チェック
- 設定例: 総合的な執筆環境
